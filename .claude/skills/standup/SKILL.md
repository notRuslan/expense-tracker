# Standup Skill

Сформируй детальный отчёт о работе за вчерашний день для стендапа.

## Контекст выполнения

Сегодняшняя дата: !`date +%Y-%m-%d`
Вчерашняя дата: !`date -d "yesterday" +%Y-%m-%d 2>/dev/null || date -v-1d +%Y-%m-%d 2>/dev/null || echo "unknown"`

Коммиты за вчера:
!`git log --after="yesterday 00:00" --before="today 00:00" --format="--- COMMIT ---%nhash: %H%nauthor: %an%ndate: %ci%nsubject: %s%nbody: %b" --stat`

Если git log не вернул результатов — попробуй расширенный диапазон:
!`git log --since="2 days ago" --until="yesterday 23:59" --format="--- COMMIT ---%nhash: %H%nauthor: %an%ndate: %ci%nsubject: %s%nbody: %b" --stat`

## Алгоритм

1. Изучи коммиты из контекста выше
2. Если коммитов нет — запусти `git log -10 --format="%ci %s"` чтобы найти ближайшие к вчерашнему дню коммиты и сообщи об этом
3. Для каждого коммита запусти `git show --stat <hash>` чтобы уточнить изменённые файлы
4. Сформируй отчёт по структуре ниже

## Структура отчёта

Выведи отчёт в формате:

```
## Стендап — <дата вчера, DD.MM.YYYY>

### Что сделано

**<Модуль/область>**
- <Понятное описание изменения без техжаргона>
- <Ещё одно изменение если есть>

**<Другой модуль>**
- <Описание>

### Детали по коммитам

| Хэш | Описание | Файлы |
|-----|----------|-------|
| `<hash короткий>` | <subject> | <список файлов> |

### Итого
- Коммитов: N
- Затронуто файлов: M
- Основные области: <перечисли scope-ы>
```

## Правила формирования отчёта

- **Язык**: русский, понятный нетехническому слушателю
- **Группировка**: по scope из Conventional Commits (backend, frontend, auth, transactions и т.д.)
- Если scope не указан — определи по изменённым файлам
- `feat` → «Добавлено: ...»
- `fix` → «Исправлено: ...»
- `refactor` → «Улучшено: ...»
- `docs` → «Задокументировано: ...»
- `chore`/`build`/`ci` → «Настроено: ...»
- Раскрывай суть изменения: не «добавить фильтр», а «теперь можно фильтровать транзакции по месяцу и году»
- Если за вчера коммитов не было — честно сообщи об этом и покажи последний коммит с датой

## Важно

- Не придумывай изменения — только то, что есть в коммитах
- Если diff большой — смотри на имена файлов и commit message, не читай весь код
