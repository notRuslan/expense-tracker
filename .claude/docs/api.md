# API — справочник эндпоинтов

Базовый URL: `http://localhost:3001`
Swagger UI: `http://localhost:3001/api/docs`

Все эндпоинты, кроме `/auth/*`, требуют заголовка:
```
Authorization: Bearer <token>
```

При ошибке валидации возвращается `400 Bad Request` с массивом `message[]`.
При невалидном / отсутствующем токене — `401 Unauthorized`.
При обращении к чужому ресурсу — `404 Not Found` (не `403`, чтобы не раскрывать существование).

---

## Auth

### `POST /auth/register`

Регистрация нового пользователя. Возвращает токен и данные пользователя.

**Тело запроса:**
```json
{
  "email": "user@example.com",
  "name": "Иван Иванов",
  "password": "secret123"
}
```

| Поле | Тип | Правила |
|---|---|---|
| `email` | `string` | валидный email |
| `name` | `string` | минимум 1 символ |
| `password` | `string` | минимум 8 символов |

**Ответ `201`:**
```json
{
  "token": "<jwt>",
  "user": {
    "id": "clx...",
    "email": "user@example.com",
    "name": "Иван Иванов",
    "createdAt": "2026-05-09T10:00:00.000Z",
    "updatedAt": "2026-05-09T10:00:00.000Z"
  }
}
```

---

### `POST /auth/login`

Вход по email и паролю. Возвращает токен и данные пользователя.

**Тело запроса:**
```json
{
  "email": "user@example.com",
  "password": "secret123"
}
```

**Ответ `200`:**
```json
{
  "token": "<jwt>",
  "user": { ... }
}
```

**`401`** — неверный email или пароль.

## Categories

Все эндпоинты изолированы по `userId` из JWT — категории других пользователей недоступны.

### `GET /categories`

Список всех категорий текущего пользователя.

**Ответ `200`:**
```json
[
  {
    "id": "clx...",
    "name": "Еда",
    "color": "#FF5733",
    "icon": "food",
    "userId": "clx...",
    "createdAt": "...",
    "updatedAt": "..."
  }
]
```

---

### `POST /categories`

Создать категорию. Имя уникально в рамках одного пользователя (`[userId, name]`).

**Тело запроса:**
```json
{
  "name": "Еда",
  "color": "#FF5733",
  "icon": "food"
}
```

| Поле | Тип | Правила |
|---|---|---|
| `name` | `string` | 1–64 символа |
| `color` | `string` | hex-цвет, формат `#RRGGBB` |
| `icon` | `string` | 1–64 символа |

**Ответ `201`:** объект `Category`.

---

### `GET /categories/:id`

Получить категорию по идентификатору.

**Ответ `200`:** объект `Category`.
**`404`** — категория не найдена или чужая.

---

### `PATCH /categories/:id`

Частично обновить категорию. Передавать только изменяемые поля.

**Тело запроса:** любое подмножество полей `CreateCategoryDto`.

**Ответ `200`:** обновлённый объект `Category`.
**`404`** — категория не найдена или чужая.

---

### `DELETE /categories/:id`

Удалить категорию. У связанных транзакций `categoryId` обнуляется (`onDelete: SetNull`).

**Ответ `204`** — без тела.
**`404`** — категория не найдена или чужая.

## Transactions

Все эндпоинты изолированы по `userId` из JWT.

### `GET /transactions`

Список транзакций пользователя (по убыванию даты) и агрегированная сводка.

**Query-параметры:**

| Параметр | Тип | Описание |
|---|---|---|
| `month` | `integer` 1–12 | Месяц фильтра. Имеет смысл только вместе с `year`. |
| `year` | `integer` 1970–9999 | Год фильтра. |

Семантика фильтра:
- Без параметров — все транзакции.
- Только `year` — все транзакции за этот год.
- `month` + `year` — за указанный месяц года.
- `month` без `year` — игнорируется.

**Ответ `200`:**
```json
{
  "items": [
    {
      "id": "clx...",
      "amount": 1500.50,
      "type": "expense",
      "description": "Покупка продуктов",
      "date": "2026-05-05T12:00:00.000Z",
      "categoryId": "clx...",
      "userId": "clx...",
      "createdAt": "...",
      "updatedAt": "..."
    }
  ],
  "summary": {
    "totalIncome": 50000.00,
    "totalExpense": 12300.75,
    "balance": 37699.25
  }
}
```

---

### `POST /transactions`

Создать транзакцию.

**Тело запроса:**
```json
{
  "amount": 1500.50,
  "type": "expense",
  "description": "Покупка продуктов",
  "date": "2026-05-05T12:00:00.000Z",
  "categoryId": "clx..."
}
```

| Поле | Тип | Правила |
|---|---|---|
| `amount` | `number` | положительное, до 2 знаков после запятой |
| `type` | `"income" \| "expense"` | обязательное |
| `description` | `string` | опционально, до 255 символов |
| `date` | `string` | ISO 8601, обязательное |
| `categoryId` | `string` | опционально; категория должна принадлежать пользователю |

**Ответ `201`:** объект `Transaction`.
**`404`** — указана чужая / несуществующая `categoryId`.

---

### `GET /transactions/:id`

Получить транзакцию по идентификатору.

**Ответ `200`:** объект `Transaction`.
**`404`** — транзакция не найдена или чужая.

---

### `PATCH /transactions/:id`

Частично обновить транзакцию. Все поля опциональны.
Передача `categoryId: null` отвязывает транзакцию от категории.

**Ответ `200`:** обновлённый объект `Transaction`.
**`404`** — транзакция не найдена/чужая или указана чужая категория.

---

### `DELETE /transactions/:id`

Удалить транзакцию.

**Ответ `204`** — без тела.
**`404`** — транзакция не найдена или чужая.

---

## Коды ответов — сводка

| Код | Когда |
|---|---|
| `200` | Успешный GET / PATCH |
| `201` | Успешный POST |
| `204` | Успешный DELETE |
| `400` | Ошибка валидации DTO |
| `401` | Нет токена / токен невалиден / пользователь удалён |
| `404` | Ресурс не найден или принадлежит другому пользователю |
| `409` | Конфликт (например, дубликат email при регистрации) |
